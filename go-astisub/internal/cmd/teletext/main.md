# 🔧 Глибокий розбір: Code Generator для Teletext Charsets в astisub

Цей файл — **генератор коду**, який створює `teletext_charsets.go` — файл з таблицями символів для підтримки телетексту в бібліотеці `astisub`. Це **build-time інструмент**, який перетворює текстові мапінги символів у оптимізований Go-код.

## 🔍 Детальний розбір ключових функцій

### 1️⃣ `extractBytes()` — ядро парсингу таблиць

```go
func extractBytes(lines [][]byte, rowCount, rowStart int) [][]byte {
    // 1. Створення 2D таблиці: [рядки][символи]
    table := make([][][]byte, rowCount)
    
    for row := rowStart; row < rowStart+rowCount; row++ {
        i := 0
        for {
            // UTF-8 декодування: один символ може бути 1-4 байти
            r, n := utf8.DecodeRune(lines[row][i:])
            if r == utf8.RuneError {
                panic("main: decoding rune error")
            }
            
            // Зберігаємо сирі байти символу
            table[row-rowStart] = append(table[row-rowStart], lines[row][i:i+n])
            i += n + 1  // +1 для пропуску роздільника (пробіл)
            
            if i >= len(lines[row])-1 {
                break
            }
        }
    }
    
    // 2. Транспонування: з [рядки][колонки] у [колонки][рядки]
    // Це потрібно, бо Teletext індексує за кодом символу, не за рядком таблиці
    var bs [][]byte
    for column := 0; column < len(table[0]); column++ {
        for row := 0; row < len(table); row++ {
            bs = append(bs, table[row][column])
        }
    }
    return bs  // 96 елементів для G0/G2 таблиць
}
```

### 🔢 Приклад роботи `extractBytes()`:

```
Вхід (chars.txt, рядки 16-17):
Рядок 16: "  ! \" £ $ % & ' ( ) * + , - . / 0 1 2 ... "
Рядок 17: "  ! \" £ $ % ژ ڤ ( ) * + ، - . / 0 1 2 ... "  // Arabic variant

Після парсингу:
table[0] = [][]byte{{0x20}, {0x21}, {0x22}, {0xc2,0xa3}, ...}  // Latin
table[1] = [][]byte{{0x20}, {0x21}, {0x22}, {0xc2,0xa3}, ...}  // Arabic variant

Транспонування (column-major order):
bs[0] = table[0][0] = {0x20}  // код 0x20 → ' '
bs[1] = table[0][1] = {0x21}  // код 0x21 → '!'
...
bs[6] = table[1][6] = {0xee,0x98,0x86}  // код 0x26 → Arabic ampersand

Результат: 96 елементів, де індекс = Teletext code - 0x20
```

> 💡 **Чому транспонування?** Teletext використовує 7-бітні коди 0x20-0x7F. Таблиця в `chars.txt` організована для зручності читання людиною (рядки), але для швидкого доступу в коді потрібен масив, де `table[code-0x20]` повертає UTF-8 байти.

```go
"FormatBytes": func(bss [][]byte) string {
    var sss []string
    for _, bs := range bss {
        var ss []string
        for _, b := range bs {
            ss = append(ss, fmt.Sprintf("0x%x", b))
        }
        sss = append(sss, fmt.Sprintf("[]byte{%s}", strings.Join(ss, ", ")))
    }
    return strings.Join(sss, ", ")
}
```

### 🗺️ Мапа `teletextCharsets`:

```go
teletextCharsets = map[uint8]map[uint8]struct {
    g0       *teletextCharset      // основна таблиця
    g2       *teletextCharset      // додаткова таблиця
    national *teletextNationalSubset // регіональні заміни
}{
    // triplet1 = 0: Latin region
    0: {
        0: {g0: teletextCharsetG0Latin, g2: teletextCharsetG2Latin, national: teletextNationalSubsetEnglish},
        1: {g0: teletextCharsetG0Latin, g2: teletextCharsetG2Latin, national: teletextNationalSubsetFrench},
        // ... інші national options
    },
    // triplet1 = 8: Arabic region
    8: {
        0: {g0: teletextCharsetG0Latin, g2: teletextCharsetG2Arabic, national: teletextNationalSubsetEnglish},
        7: {g0: teletextCharsetG0Arabic, g2: teletextCharsetG2Arabic},  // Pure Arabic!
    },
    // triplet1 = 4: Cyrillic region
    4: {
        0: {g0: teletextCharsetG0CyrillicOption1, g2: teletextCharsetG2Cyrillic},
        // ... інші варіанти
    },
}
```

> ⚠️ **Критично для арабської**: Використовуйте `triplet1=8, nationalOption=7` для "Pure Arabic" режиму, де і G0, і G2 — арабські таблиці.

## ✅ Ваш use-case: інтеграція у CCTV HLS Processor

### Сценарій 1: Динамічний вибір charset для каналу

```go
// ChannelConfig — конфігурація каналу
type ChannelConfig struct {
    ChannelID          string
    LanguageCode       string  // "ara", "rus", "eng"
    TeletextPage       int
    TeletextPID        uint16
    CharacterTableID   uint16  // 12338 = Arabic, 12337 = Cyrillic
}

// GetTeletextCharset — вибір таблиці за конфігурацією
func (cfg *ChannelConfig) GetTeletextCharset() (*teletextCharset, *teletextCharset, *teletextNationalSubset) {
    // Визначаємо triplet1 за мовою
    triplet1 := uint8(0)  // default: Latin
    switch cfg.LanguageCode {
    case "ara", "ar":
        triplet1 = 8  // Arabic region
    case "rus", "ru", "ukr", "uk":
        triplet1 = 4  // Cyrillic region
    case "ell", "el", "grc":
        triplet1 = 6  // Greek region
    }
    
    // Визначаємо national option (зазвичай 7 для "pure" мови)
    nationalOption := uint8(7)
    
    // Отримуємо таблиці з мапи
    if region, ok := teletextCharsets[triplet1]; ok {
        if charset, ok := region[nationalOption]; ok {
            return charset.g0, charset.g2, charset.national
        }
    }
    
    // Fallback: Latin
    return teletextCharsetG0Latin, teletextCharsetG2Latin, nil
}
```

### Сценарій 2: Декодування Teletext байтів у текст

```go
// TeletextDecoder — декодер для одного каналу
type TeletextDecoder struct {
    g0, g2       *teletextCharset
    national     *teletextNationalSubset
    g2Active     bool
    accumulator  *byte  // для діакритики (якщо потрібно)
}

// DecodeByte — декодування одного байта у UTF-8
func (d *TeletextDecoder) DecodeByte(b byte) []byte {
    // Control codes (0x00-0x1F) — не декодуємо
    if b < 0x20 {
        return nil
    }
    
    // Індекс у таблиці: код - 0x20
    idx := int(b - 0x20)
    if idx < 0 || idx >= 96 {
        return nil
    }
    
    // Спочатку перевіряємо national subset (якщо є)
    if d.national != nil {
        for posIdx, pos := range teletextNationalSubsetCharactersPositionInG0 {
            if pos == b {
                return d.national[posIdx]
            }
        }
    }
    
    // Потім G0 або G2 залежно від стану
    if d.g2Active {
        return d.g2[idx]
    }
    return d.g0[idx]
}

// ToggleG2 — перемикання між G0/G2 (через control code 0x0F/0x0E)
func (d *TeletextDecoder) ToggleG2(active bool) {
    d.g2Active = active
}
```

### Сценарій 3: Обробка розривів у потоці

```go
// OnSegmentBoundary — скидання стану між сегментами
func (d *TeletextDecoder) OnSegmentBoundary() {
    // Скидаємо G2 стан та accumulator
    d.g2Active = false
    d.accumulator = nil
    
    // ⚠️ Не скидаємо самі таблиці — вони immutable і кешуються на рівні каналу
}

// ProcessTeletextRow — обробка одного рядка телетексту
func (d *TeletextDecoder) ProcessTeletextRow(row []byte) string {
    var textBuilder strings.Builder
    
    for _, b := range row {
        // Перевірка control codes
        switch b {
        case 0x0F:  // Activate G2
            d.ToggleG2(true)
            continue
        case 0x0E:  // Activate G0
            d.ToggleG2(false)
            continue
        }
        
        // Декодування символу
        if utf8Bytes := d.DecodeByte(b); utf8Bytes != nil {
            textBuilder.Write(utf8Bytes)
        }
    }
    
    return textBuilder.String()
}
```

## 🐞 Поширені проблеми та рішення

| Проблема | Причина | Рішення |
|----------|---------|---------|
| **"????" замість арабського тексту** | Неправильний `triplet1` або `nationalOption` | Використовуйте `triplet1=8, nationalOption=7` для Arabic |
| **Діакритика "з'їдає" наступний символ** | Не реалізовано accumulator для діакритики | Додайте логіку як у `stlCharacterHandler.decode()` |
| **Таблиця не знайдена в мапі** | `triplet1` або `nationalOption` поза діапазоном | Додайте fallback на Latin таблиці |
| **Неправильний порядок символів** | Помилка в `extractBytes()` транспонуванні | Перевірте, що `column` зовнішній цикл, `row` — внутрішній |
| **UTF-8 помилка при парсингу `chars.txt`** | Файл не у валідному UTF-8 | Переконайтеся, що `chars.txt` збережено у UTF-8 без BOM |

### 2. Пакетне декодування рядків:

```go
// Замість індивідуального DecodeByte для кожного байта:
func (d *TeletextDecoder) DecodeRow(row []byte) []byte {
    // Попередньо виділяємо буфер (макс. 4 байти UTF-8 на символ × 96 символів)
    result := make([]byte, 0, 384)
    
    for _, b := range row {
        if b >= 0x20 && b <= 0x7F {
            idx := int(b - 0x20)
            
            // National subset check (оптимізовано)
            if d.national != nil {
                // Прямий доступ за індексом замість циклу
                if posIdx := nationalSubsetIndex[b]; posIdx >= 0 {
                    result = append(result, d.national[posIdx]...)
                    continue
                }
            }
            
            // G0/G2 вибір
            table := d.g0
            if d.g2Active {
                table = d.g2
            }
            result = append(result, table[idx]...)
        }
    }
    return result
}
```

> 💡 **Ключова рекомендація для вашого проекту**: Оскільки ви працюєте з **арабською мовою в реальному часі**:
> 1. **Завжди використовуйте `triplet1=8, nationalOption=7`** для Arabic — це гарантує правильну G0 таблицю.
> 2. **Кешуйте `TeletextDecoder` на рівні каналу** — таблиці immutable, не створюйте нові для кожного сегменту.
> 3. **Скидайте `g2Active` та `accumulator` між сегментами** — control codes не повинні "перетікати" між сегментами.
> 4. **Додайте валідацію довжини таблиць** у `init()` — це допоможе виявити помилки генерації на етапі компіляції.
> 5. **Тестуйте round-trip** (Teletext bytes → decoder → UTF-8 → encoder → Teletext bytes) для вашого каналу, щоб переконатися, що арабський текст зберігається коректно.
