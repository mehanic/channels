# 📦 `Udta3GppString`: Метадані 3GPP у MP4-файлах

Це код для роботи з **текстовими метаданими** у форматі **3GPP** (стандарт для мобільного мультимедіа), які зберігаються всередині боксу `udta` (User Data) MP4-файлу.

---

## 🎯 Коротка відповідь

> **Це "контейнер" для текстових метаданих** типу: назва передачі, опис, автор, жанр, мова — у форматі, сумісному з мобільними плеєрами та стандартом 3GPP.

## 🧱 Розбір коду по частинах

### 🔹 1. Список типів боксів для метаданих

```go
var udta3GppMetaBoxTypes = []BoxType{
	StrToBoxType("titl"),  // Title — назва
	StrToBoxType("dscp"),  // Description — опис
	StrToBoxType("cprt"),  // Copyright — авторські права
	StrToBoxType("perf"),  // Performer — виконавець
	StrToBoxType("auth"),  // Author — автор
	StrToBoxType("gnre"),  // Genre — жанр
}
```

| Тип боксу | Призначення | Приклад значення |
|-----------|-------------|-----------------|
| `titl` | Назва передачі/відео | `"Ранкові новини"`, `"Погода"` |
| `dscp` | Опис контенту | `"Огляд подій за тиждень"` |
| `cprt` | Авторські права | `"© ТРК «Студія 1+1», 2026"` |
| `perf` | Виконавець/ведучий | `"Іван Петренко"` |
| `auth` | Автор сценарію | `"Марія Коваленко"` |
| `gnre` | Жанр | `"Новини"`, `"Документальний"`, `"Розваги"` |

> 🎯 **Навіщо список?** Щоб зареєструвати одну структуру `Udta3GppString` для **всіх** цих типів боксів — не потрібно писати 6 окремих структур.

### 🔹 2. Реєстрація в `init()` — магія бібліотеки

```go
func init() {
	for _, bt := range udta3GppMetaBoxTypes {
		AddAnyTypeBoxDefEx(&Udta3GppString{}, bt, isUnderUdta, 0)
	}
}
```

| Параметр | Значення | Пояснення |
|----------|----------|-----------|
| `&Udta3GppString{}` | екземпляр структури | "Ось шаблон для парсингу" |
| `bt` | тип боксу (`titl`, `dscp`...) | "Застосуй цей шаблон до боксу типу..." |
| `isUnderUdta` | функція-умова | "Тільки якщо ми всередині `udta` боксу" |
| `0` | версія боксу | `FullBox` версія 0 (стандартна) |

**Що робить `AddAnyTypeBoxDefEx`?**
```
🔹 Реєструє в бібліотеці: 
   "Якщо зустрічається бокс типу 'titl' ВСЕРЕДИНІ 'udta' → парсити його як Udta3GppString"

🔹 Це дозволяє одному коду обробляти 6 різних типів боксів!
```

### 🔹 3. Структура `Udta3GppString` — розбір тегів `mp4:"..."`

```go
type Udta3GppString struct {
	AnyTypeBox                    // ← Базовий тип для "будь-якого" боксу
	FullBox  `mp4:"0,extend"`     // ← Розширення: version + flags (стандарт ISO)
	Pad      bool    `mp4:"1,size=1,hidden"`                    // ← 1 біт падінгу (ігнорується)
	Language [3]byte `mp4:"2,size=5,iso639-2"`                  // ← 3×5 біт = код мови
	Data     []byte  `mp4:"3,size=8,string"`                    // ← Текст метаданих
}
```

#### 📐 Візуалізація бітового формату:

```
📦 Бокс "titl" (приклад: назва "Новини" українською)
┌─────────────────────────────────┐
│ Заголовок боксу (8 байт)        │
│ [00 00 00 0E] [74 69 74 6C]    │ ← size=14, type="titl"
├─────────────────────────────────┤
│ FullBox: version + flags (4B)   │
│ [00] [00 00 00]                 │ ← version=0, flags=0
├─────────────────────────────────┤
│ Pad (1 біт)                     │
│ [0].... ...                     │ ← ігнорується (заповнювач)
├─────────────────────────────────┤
│ Language (3×5 біт = 15 біт)     │
│ [11001] [01011] [10010]         │ ← "ukr" в ISO-639-2/T кодуванні
├─────────────────────────────────┤
│ Data (рядок, 8 біт/символ)      │
│ [4E 6F 76 69 6E 79]             │ ← "Новини" в UTF-8
└─────────────────────────────────┘
```

### 🔹 Детальний розбір тегів

#### ✅ `mp4:"0,extend"` — розширення `FullBox`
```go
FullBox  `mp4:"0,extend"`
```
| Атрибут | Значення |
|---------|----------|
| `0` | Порядок поля #0 у серіалізації |
| `extend` | "Встав сюди поля `Version` та `Flags` з `FullBox`" |

> 🎯 **Результат**: Структура автоматично отримує стандартні 4 байти заголовка (`version` + `flags`).

---

#### ✅ `mp4:"1,size=1,hidden"` — прихований падінг
```go
Pad bool `mp4:"1,size=1,hidden"`
```
| Атрибут | Значення |
|---------|----------|
| `1` | Поле #1 (після `FullBox`) |
| `size=1` | 1 біт |
| `hidden` | "Не записуй це у файл, тільки читай/ігноруй" |

> 🎯 **Навіщо?** Стандарт 3GPP вимагає вирівнювання на байт. Після 1 біта `Pad` + 15 біт `Language` = 16 біт = 2 байти (ідеально!).

---

#### ✅ `mp4:"2,size=5,iso639-2"` — код мови
```go
Language [3]byte `mp4:"2,size=5,iso639-2"`
```
| Атрибут | Значення |
|---------|----------|
| `2` | Поле #2 |
| `size=5` | **Кожен** елемент масиву — 5 біт |
| `iso639-2` | Спеціальне кодування: `код = (b1-'a'+1)<<10 \| (b2-'a'+1)<<5 \| (b3-'a'+1)` |

**Приклад: "ukr" (українська)**:
```
🔤 Літера → код:
   'u' = 21-я літера алфавіту → 20 (0-indexed) → 10100 (5 біт)
   'k' = 11-я → 10 → 01010
   'r' = 18-я → 17 → 10001

🔢 Разом у 15 бітах:
   10100 01010 10001 = 0x14A9 = 5289 (десяткове)

📦 У файлі це зберігається як 2 байти:
   [00 01 0100 1010 1000 1] → [05] [29] (при вирівнюванні)
```

> ⚠️ **Важливо**: Це **НЕ** звичайні ASCII-байти! Це спеціальне 5-бітне кодування, де `'a'=1`, `'b'=2`, ..., `'z'=26`.

---

#### ✅ `mp4:"3,size=8,string"` — текстові дані
```go
Data []byte `mp4:"3,size=8,string"`
```
| Атрибут | Значення |
|---------|----------|
| `3` | Поле #3 |
| `size=8` | Кожен байт = 8 біт (стандартний байт) |
| `string` | "Інтерпретуй ці байти як рядок (не як числа)" |

**Приклад: "Новини" в UTF-8**:
```
🔤 "Новини" → UTF-8 байти:
   Н = 0xD0 0x9D
   о = 0xD0 0xBE  
   в = 0xD0 0xB2
   и = 0xD0 0xB8
   н = 0xD0 0xBD
   і = 0xD1 0x96

📦 У файлі: [D0 9D D0 BE D0 B2 D0 B8 D0 BD D1 96] (12 байт)
```

> ✅ Бібліотека автоматично читає до кінця боксу або до наступного поля.

## 🛠️ Практичне використання у вашому HLS-процесорі

### 🔹 Приклад 1: Додавання назви передачі до сегмента

```go
import "github.com/abema/go-mp4"

func addTitleToSegment(filePath, title, langCode string) error {
	// 1. Створити структуру метаданих
	meta := &mp4.Udta3GppString{
		FullBox: mp4.FullBox{Version: 0, Flags: [3]byte{0, 0, 0}},
		Language: encodeISO639_2(langCode), // "ukr" → [3]byte{20, 10, 17}
		Data:     []byte(title),             // "Ранкові новини"
	}
	
	// 2. Відкрити файл для дописування
	f, err := os.OpenFile(filePath, os.O_RDWR, 0644)
	if err != nil { return err }
	defer f.Close()
	
	// 3. Знайти або створити бокс "udta"
	_, err = mp4.ReadBoxStructure(f, func(h *mp4.ReadHandle) (interface{}, error) {
		if h.BoxInfo.Type == mp4.BoxTypeUdta() {
			// 4. Записати бокс "titl" всередині "udta"
			titleInfo := &mp4.BoxInfo{
				Type: mp4.StrToBoxType("titl"),
				Size: 16 + uint64(len(meta.Data)), // приблизний розмір
			}
			updated, _ := mp4.WriteBoxInfo(f, titleInfo)
			meta.Marshal(f)  // серіалізувати дані
		}
		return nil, nil
	})
	
	return err
}

// Допоміжна функція: кодування мови в ISO-639-2/T формат
func encodeISO639_2(lang string) [3]byte {
	if len(lang) < 3 { return [3]byte{0, 0, 0} }
	var result [3]byte
	for i := 0; i < 3; i++ {
		// 'a'=1, 'b'=2, ..., 'z'=26 → 5 біт
		result[i] = byte(lang[i] - 'a' + 1)
	}
	return result
}
```

### 🔹 Приклад 2: Читання метаданих з існуючого сегмента

```go
func readMetadata(filePath string) (map[string]string, error) {
	result := make(map[string]string)
	
	f, err := os.Open(filePath)
	if err != nil { return nil, err }
	defer f.Close()
	
	_, err = mp4.ReadBoxStructure(f, func(h *mp4.ReadHandle) (interface{}, error) {
		// Перевіряємо, чи це один з наших типів метаданих
		boxType := h.BoxInfo.Type.String()
		if boxType == "titl" || boxType == "dscp" || boxType == "cprt" {
			meta := &mp4.Udta3GppString{}
			if _, err := h.ReadPayload(meta); err != nil {
				return nil, err
			}
			
			// Декодуємо мову (спрощено)
			lang := decodeISO639_2(meta.Language)
			
			// Зберігаємо результат
			result[boxType] = fmt.Sprintf("[%s] %s", lang, string(meta.Data))
		}
		return nil, nil
	})
	
	return result, err
}

// Спрощене декодування (для прикладу)
func decodeISO639_2(code [3]byte) string {
	result := make([]byte, 3)
	for i := 0; i < 3; i++ {
		result[i] = byte(code[i] + 'a' - 1)
	}
	return string(result)
}
```

## 🔍 Чому саме такий формат? (3GPP специфіка)

### 📐 Історичний контекст:
```
📱 2000-ні: мобільні телефони з обмеженою пам'яттю
📡 3GPP стандарт: оптимізація кожного біта
🔤 Мова кодується в 15 бітах замість 24 (3 байти ASCII)
   → Економія 37.5% на кожному рядку метаданих!
```

### 🔄 Порівняння форматів:

| Формат | "ukr" у байтах | Переваги | Недоліки |
|--------|---------------|----------|----------|
| **ISO-639-2/T (3GPP)** | 2 байти (15 біт + падінг) | ✅ Економія місця, стандарт для мобільних | ❌ Складніше кодування |
| **ASCII/UTF-8** | 3 байти ("u","k","r") | ✅ Просто читати людиною | ❌ Більший розмір |
| **BCP-47 ("uk-UA")** | 5+ байт | ✅ Сучасний стандарт, регіон | ❌ Занадто великий для 3GPP |

> 🎯 **Висновок**: 3GPP-формат — це компроміс між розміром та функціональністю для мобільних мереж 2000-х. Сьогодні ми його використовуємо для **сумісності** зі старими плеєрами та стандартами.

## ⚠️ Поширені помилки та як їх уникнути

| Помилка | Симптоми | Рішення |
|---------|----------|---------|
| Неправильне кодування мови | Плеєр показує "und" (undefined) замість "ukr" | Використовуйте `encodeISO639_2()`, не просто `[]byte("ukr")` |
| Забути `AnyTypeBox` | `panic: field not found` при парсингу | Завжди вбудовуйте `AnyTypeBox` у структуру для `AddAnyTypeBoxDefEx` |
| Неправильний порядок полів | Дані зсуваються, парсинг ламається | Поля мають йти строго у порядку `mp4:"0,...", mp4:"1,..."` |
| Ігнорування `hidden` | Зайві байти у файлі | Поля з `hidden` не записуються — тільки читаються для вирівнювання |
| Не перевіряти контекст | Парсинг `titl` поза `udta` як метадані | Завжди використовуйте умову `isUnderUdta` при реєстрації |

## ❓ Часті питання

**Q: Чи можу я використовувати UTF-16 замість UTF-8 у `Data`?**  
A: Технічно — так, але 3GPP стандарт рекомендує UTF-8. Деякі старі плеєри можуть не підтримувати UTF-16.

**Q: Як додати власний тип метаданих (наприклад, "час ефіру")?**  
```go
// 1. Оголосіть новий тип боксу
func BoxTypeAirTime() mp4.BoxType { return mp4.StrToBoxType("airt") }

// 2. Додайте у список реєстрації
var udta3GppMetaBoxTypes = []mp4.BoxType{
	// ... існуючі ...
	BoxTypeAirTime(),  // ← новий тип
}

// 3. Використовуйте ту саму структуру Udta3GppString
//    (вона універсальна для будь-якого текстового метаданого)
```

**Q: Чому `Data` — це `[]byte`, а не `string`?**  
A: У бінарному форматі немає поняття "рядок" — тільки байти. `string` у Go — це абстракція. Використання `[]byte` дає контроль над кодуванням (UTF-8, ASCII, тощо).

**Q: Як перевірити, чи метадані коректно записані?**  
```bash
# Використайте ffprobe (з FFmpeg):
ffprobe -show_frames -print_format json segment.m4s | grep -i title

# Або mp4box (з GPAC):
mp4box -info segment.m4s | grep -A5 "User Data"
```
