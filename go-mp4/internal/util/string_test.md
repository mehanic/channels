### 🔹 `TestFormatSignedFixedFloat1616` — тест форматування знакових 16.16 чисел

```go
func TestFormatSignedFixedFloat1616(t *testing.T) {
    // 🔹 ТЕСТ 1: Ціле позитивне значення
    // 0x00010000 = 65536 → 65536/65536 = 1.0 → нижні 16 біт = 0 → "1"
    assert.Equal(t, "1", FormatSignedFixedFloat1616(0x00010000))
    
    // 🔹 ТЕСТ 2: Велике ціле позитивне
    // 0x04d20000 = 1234 << 16 → 1234.0 → "1234"
    assert.Equal(t, "1234", FormatSignedFixedFloat1616(0x04d20000))
    
    // 🔹 ТЕСТ 3: Ціле від'ємне значення
    // -0x04d20000 → -1234.0 → "-1234"
    assert.Equal(t, "-1234", FormatSignedFixedFloat1616(-0x04d20000))
    
    // 🔹 ТЕСТ 4: Дробове позитивне (1.5)
    // 0x00018000 = 65536 + 32768 = 98304 → 98304/65536 = 1.5
    // нижні 16 біт ≠ 0 → "1.50000" (5 знаків після коми)
    assert.Equal(t, "1.50000", FormatSignedFixedFloat1616(0x00018000))
    
    // 🔹 ТЕСТ 5: Дробове з більшою точністю
    // 0x04d29161 = 1234*65536 + 37217 = 80871425
    // 80871425/65536 ≈ 1234.567886... → округлено до "1234.56789"
    assert.Equal(t, "1234.56789", FormatSignedFixedFloat1616(0x04d29161))
    
    // 🔹 ТЕСТ 6: Від'ємне дробове
    // -0x04d29161 → "-1234.56789"
    assert.Equal(t, "-1234.56789", FormatSignedFixedFloat1616(-0x04d29161))
}
```

**🎯 Що тестується:**

| Вхід (hex) | Десяткове значення | Очікуваний вивід | Чому це важливо |
|------------|-------------------|-----------------|----------------|
| `0x00010000` | 1.0 | `"1"` | ✅ Цілі значення без зайвих нулів |
| `0x04d20000` | 1234.0 | `"1234"` | ✅ Великі цілі числа |
| `-0x04d20000` | -1234.0 | `"-1234"` | ✅ Коректна обробка знаку |
| `0x00018000` | 1.5 | `"1.50000"` | ✅ Дробові значення з 5 знаками |
| `0x04d29161` | ≈1234.56789 | `"1234.56789"` | ✅ Точне округлення дробової частини |
| `-0x04d29161` | ≈-1234.56789 | `"-1234.56789"` | ✅ Знак + дробова частина |

**🔑 Ключова логіка перевірки:**
```
🔹 Алгоритм форматування 16.16:
1. Перевірити нижні 16 біт: val & 0xffff
2. Якщо == 0 → ціле число → повернути val >> 16 як int
3. Якщо != 0 → дробове → повернути float64(val)/65536 з 5 знаками

🔢 Приклад для 0x00018000:
• val = 0x00018000 = 98304
• val & 0xffff = 0x8000 = 32768 ≠ 0 → дробове
• float64(98304) / 65536.0 = 1.5
• FormatFloat(1.5, 'f', 5, 64) = "1.50000" ✅
```

### 🔹 `TestFormatSignedFixedFloat88` — тест 8.8 fixed-point формату

```go
func TestFormatSignedFixedFloat88(t *testing.T) {
    // 🔹 Ціле позитивне: 0x0100 = 256 → 256/256 = 1.0 → "1"
    assert.Equal(t, "1", FormatSignedFixedFloat88(0x0100))
    
    // 🔹 Велике ціле: 0x7b00 = 123<<8 = 31488 → 31488/256 = 123 → "123"
    assert.Equal(t, "123", FormatSignedFixedFloat88(0x7b00))
    
    // 🔹 Від'ємне ціле: -0x7b00 → "-123"
    assert.Equal(t, "-123", FormatSignedFixedFloat88(-0x7b00))
    
    // 🔹 Дробове 1.5: 0x0180 = 256+128 = 384 → 384/256 = 1.5 → "1.500" (3 знаки)
    assert.Equal(t, "1.500", FormatSignedFixedFloat88(0x0180))
    
    // 🔹 Дробове з точністю: 0x7b75 = 123*256 + 117 = 31605
    // 31605/256 ≈ 123.45703125 → округлено до "123.457"
    assert.Equal(t, "123.457", FormatSignedFixedFloat88(0x7b75))
    
    // 🔹 Від'ємне дробове: -0x7b75 → "-123.457"
    assert.Equal(t, "-123.457", FormatSignedFixedFloat88(-0x7b75))
}
```

**🎯 Що тестується:**

| Вхід (hex) | Формат 8.8 | Очікуваний вивід | Чому це важливо |
|------------|-----------|-----------------|----------------|
| `0x0100` | 1.0 | `"1"` | ✅ Цілі значення без зайвих нулів |
| `0x7b00` | 123.0 | `"123"` | ✅ Великі цілі для 8.8 формату |
| `0x0180` | 1.5 | `"1.500"` | ✅ Дробові з 3 знаками (менша точність ніж 16.16) |
| `0x7b75` | ≈123.457 | `"123.457"` | ✅ Точне округлення для 8.8 формату |

**🔑 Відмінності 8.8 від 16.16:**
| Аспект | 16.16 | 8.8 |
|--------|-------|-----|
| Розмір | 32 біти | 16 біт |
| Дільник | 65536 (2^16) | 256 (2^8) |
| Знаків після коми | 5 | 3 |
| Тип у Go | `int32`/`uint32` | `int16` |
| Точність | ~0.000015 | ~0.0039 |

### 🔹 `TestEscapeUnprintables` — тест очищення рядків

```go
func TestEscapeUnprintables(t *testing.T) {
    // 🔹 Вхідний масив байт з різними типами символів:
    input := []byte{
        0x00,             // 🔹 NULL (керуючий) → '.'
        0x41, 0x42, 0x43, // 🔹 "ABC" (друковані ASCII) → "ABC"
        0x0a,             // 🔹 LF \n (керуючий) → '.'
        0xe3, 0x81, 0x82, // 🔹 "あ" (UTF-8, друкований) → "あ"
        0xe3, 0x81, 0x84, // 🔹 "い" (UTF-8, друкований) → "і"
        0xe3, 0x81, 0x86, // 🔹 "う" (UTF-8, друкований) → "у"
        0x0d,             // 🔹 CR \r (керуючий) → '.'
    }
    
    // 🔹 Очікуваний результат:
    // "." + "ABC" + "." + "あいう" + "." = ".ABC.あいう."
    assert.Equal(t, ".ABC.あいう.", EscapeUnprintables(string(input)))
}
```

**🎯 Що тестується:**

| Вхідний байт | Символ | `unicode.IsGraphic` | Результат | Чому це важливо |
|-------------|--------|-------------------|-----------|----------------|
| `0x00` | NULL | ❌ false | `.` | ✅ Керуючі символи замінюються |
| `0x41-43` | "ABC" | ✅ true | "ABC" | ✅ Друковані ASCII зберігаються |
| `0x0a` | \n (LF) | ❌ false | `.` | ✅ Переноси рядка замінюються |
| `0xe3,0x81,0x82` | "あ" | ✅ true | "あ" | ✅ Unicode-символи зберігаються |
| `0x0d` | \r (CR) | ❌ false | `.` | ✅ Повернення каретки замінюється |

**🔑 Ключова логіка `EscapeUnprintable`:**
```
🔹 Алгоритм:
1. Для кожного rune у рядку:
   • Якщо unicode.IsGraphic(r) == true → повернути r як є
   • Інакше → повернути '.' (замінник)
2. strings.Map застосовує цю функцію до всього рядка

🔢 Приклад для "Hello\x00World":
• 'H','e','l','l','o' → IsGraphic=true → зберігаються
• '\x00' (0x00) → IsGraphic=false → замінюється на '.'
• 'W','o','r','l','d' → зберігаються
• Результат: "Hello.World" ✅
```

### 🔹 Приклад 3: Валідація fixed-point значень перед записом

```go
// 🔹 Функція для конвертації float у 16.16 fixed-point з валідацією
func floatToFixed1616(val float64) (int32, error) {
    // 🔹 Перевірка діапазону: ±32768 для цілої частини
    if val > 32768.0 || val < -32768.0 {
        return 0, fmt.Errorf("value %.2f out of range for 16.16", val)
    }
    
    // 🔹 Конвертація: val * 65536
    fixed := int32(val * (1 << 16))
    
    // 🔹 Перевірка на втрату точності
    reconstructed := float64(fixed) / (1 << 16)
    if math.Abs(reconstructed-val) > 1e-5 {
        log.Printf("⚠️  Precision loss: %.6f → %d (16.16)", val, fixed)
    }
    
    return fixed, nil
}

// 🔹 Використання при створенні боксу:
func createTkhd(width, height float64) (*mp4.Tkhd, error) {
    widthFixed, err := floatToFixed1616(width)
    if err != nil {
        return nil, fmt.Errorf("invalid width: %w", err)
    }
    
    heightFixed, err := floatToFixed1616(height)
    if err != nil {
        return nil, fmt.Errorf("invalid height: %w", err)
    }
    
    return &mp4.Tkhd{
        Width:  widthFixed,
        Height: heightFixed,
    }, nil
}
```

## ⚠️ Поширені помилки та як їх уникнути

| Помилка | Симптоми | Рішення |
|---------|----------|---------|
| Неправильне перетворення fixed-point | "1920.00000" замість "1920" для цілих | Перевіряйте `val&0xffff == 0` перед форматуванням |
| Втрата точності при конвертації | `1.00001` стає `1.0` через округлення | Використовуйте достатню точність: 5 знаків для 16.16, 3 для 8.8 |
| Заміна корисних символів на `.` | "Line1\nLine2" → "Line1.Line2" (втрата структури) | Для логування використовуйте `EscapeUnprintables`, для збереження — ні |
| Ігнорування знаку у signed fixed-point | Від'ємні значення відображаються як додатні | Використовуйте `FormatSignedFixedFloat*` для signed, `Unsigned` для unsigned |
| Переповнення при конвертації | `float64(val)/(1<<16)` дає невірний результат для великих `val` | Перевіряйте діапазон вхідних значень перед конвертацією |

## 📋 Чекліст для вашого проекту

```
[ ] При форматуванні fixed-point чисел:
    • Використовуйте FormatSignedFixedFloat1616 для полів типу Width/Height/Matrix
    • Перевіряйте, чи значення ціле (val&0xffff==0) для кращої читабельності
    • Для 8.8 форматів використовуйте FormatSignedFixedFloat88 з 3 знаками після коми
    • Тестуйте граничні значення: 0, ±цілі, ±дробові, максимальні/мінімальні

[ ] При роботі з текстовими полями:
    • Використовуйте EscapeUnprintables для безпечного виводу у логах/терміналі
    • Не використовуйте очищення для збереження даних — тільки для відображення
    • Логувайте попередження, якщо знайдено недрюковані символи у важливих полях
    • Тестуйте з різними кодуваннями: ASCII, UTF-8, кирилиця, емодзі

[ ] Для валідації вхідних даних:
    • Перевіряйте діапазон float перед конвертацією у fixed-point
    • Попереджайте про втрату точності при округленні
    • Тестуйте крайні випадки: 0, ±цілі, ±дробові, граничні значення

[ ] Для дебагу:
    • Логувайте сирі fixed-point значення у hex: log.Printf("🔢 Width raw: 0x%08x", val)
    • Порівнюйте форматований вивід з очікуваним: "1920" vs "1920.00000"
    • Тестуйте EscapeUnprintables з різними наборами символів: \0, \n, \t, Unicode

[ ] Для тестування:
    • Створюйте тестові випадки для fixed-point: цілі, дробові, від'ємні, граничні
    • Перевіряйте round-trip: float → fixed → formatted float → порівняння
    • Тестуйте EscapeUnprintables з контрольованими вхідними даними
```
