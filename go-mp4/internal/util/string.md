# 📐 `util`: Утиліти для форматування чисел та очищення рядків у `go-mp4`

Це **допоміжний модуль** бібліотеки `go-mp4`, який надає функції для **форматування фіксовано-крапкових чисел** (fixed-point) та **очищення рядків від недрюкованих символів** — критично для людино-читабельного виводу метаданих у інструментах `dump`, `probe` та `stringify`.

---

## 🎯 Коротка відповідь

> **Це "форматувальник" для специфічних типів даних MP4**: він перетворює фіксовано-крапкові числа (16.16, 8.8) у зручні десяткові рядки та замінює недрюковані символи на `.` для безпечного виводу текстових полів.

## 🧱 Основні функції

### 🔹 `FormatSignedFixedFloat1616` — форматування 32-бітного знакового fixed-point (16.16)

```go
func FormatSignedFixedFloat1616(val int32) string {
    // 🔹 Перевірка: чи є дробова частина нульовою?
    if val&0xffff == 0 {  // 🔹 Маска 0xffff = нижні 16 біт
        // ✅ Ціле число: повертаємо старші 16 біт як int
        return strconv.Itoa(int(val >> 16))
    } else {
        // ❌ Є дробова частина: конвертуємо у float
        // Формула: val / 2^16 = val / 65536
        return strconv.FormatFloat(float64(val)/(1<<16), 'f', 5, 64)
    }
}
```

**🎯 Призначення**: Перетворити **16.16 fixed-point** число (стандарт для полів типу `Matrix`, `Width`, `Height` у боксах `tkhd`, `mvhd`) у людино-читабельний рядок.

**🔢 Формат 16.16:**
```
📐 32 біти: [16 біт ціла частина][16 біт дробова частина]
• Знак: старший біт цілої частини (для signed)
• Діапазон: ±32768.99998 (для signed)

🔢 Приклади:
• 0x00010000 = 1.0 → "1"
• 0x00018000 = 1.5 → "1.50000"
• 0xffff8000 = -0.5 → "-0.50000" (для signed)
• 0x00000001 = 0.00001525... → "0.00002" (округлено до 5 знаків)
```

**🔄 Алгоритм:**
```
🔹 Вхід: val = 0x00018000 (1.5 у 16.16)
│
▼
🔹 Перевірка дробової частини:
   • val & 0xffff = 0x8000 != 0 → є дробова частина
│
▼
🔹 Конвертація у float:
   • float64(0x00018000) = 98304.0
   • 98304.0 / 65536.0 = 1.5
   • FormatFloat(1.5, 'f', 5, 64) = "1.50000"
│
▼
🔹 Вихід: "1.50000"
```

**🎯 Навіщо перевірка `val&0xffff == 0`?**
- ✅ Для цілих значень (напр. `Width=1920.0`) виводити `"1920"`, а не `"1920.00000"` — краща читабельність
- ✅ Уникнення зайвих нулів у виводі для поширених випадків

### 🔹 `FormatUnsignedFixedFloat1616` — те саме для беззнакових чисел

```go
func FormatUnsignedFixedFloat1616(val uint32) string {
    // 🔹 Логіка ідентична, але для uint32
    if val&0xffff == 0 {
        return strconv.Itoa(int(val >> 16))
    } else {
        return strconv.FormatFloat(float64(val)/(1<<16), 'f', 5, 64)
    }
}
```

**🎯 Призначення**: Форматувати **беззнакові 16.16 fixed-point** числа (напр. у боксах, де від'ємні значення не мають сенсу).

**🔢 Приклади:**
```
• 0x00020000 = 2.0 → "2"
• 0x00024000 = 2.25 → "2.25000"
• 0x00008000 = 0.5 → "0.50000"
```

### 🔹 `FormatSignedFixedFloat88` — форматування 16-бітного знакового fixed-point (8.8)

```go
func FormatSignedFixedFloat88(val int16) string {
    // 🔹 Перевірка: чи є дробова частина нульовою?
    if val&0xff == 0 {  // 🔹 Маска 0xff = нижні 8 біт
        // ✅ Ціле число: повертаємо старші 8 біт як int
        return strconv.Itoa(int(val >> 8))
    } else {
        // ❌ Є дробова частина: конвертуємо у float
        // Формула: val / 2^8 = val / 256
        return strconv.FormatFloat(float64(val)/(1<<8), 'f', 3, 32)
    }
}
```

**🎯 Призначення**: Перетворити **8.8 fixed-point** число (менш поширений формат, але використовується у деяких полях) у людино-читабельний рядок.

**🔢 Формат 8.8:**
```
📐 16 біт: [8 біт ціла частина][8 біт дробова частина]
• Діапазон: ±128.996 (для signed)
• Точність: ~0.0039 (1/256)

🔢 Приклади:
• 0x0100 = 1.0 → "1"
• 0x0180 = 1.5 → "1.500"
• 0xff80 = -0.5 → "-0.500" (для signed)
```

**🔑 Відмінності від 16.16:**
| Аспект | 16.16 | 8.8 |
|--------|-------|-----|
| Розмір | 32 біти | 16 біт |
| Точність | 1/65536 ≈ 0.000015 | 1/256 ≈ 0.0039 |
| Знаків після коми у виводі | 5 | 3 |
| Тип у Go | `int32`/`uint32` | `int16` |

### 🔹 `EscapeUnprintable` — заміна недрюкованого символу на `.`

```go
func EscapeUnprintable(r rune) rune {
    // 🔹 Перевірка: чи є символ "графічним" (друкованим)?
    if unicode.IsGraphic(r) {
        return r  // ✅ Повертаємо як є
    }
    // ❌ Недрюкований: замінюємо на '.'
    return rune('.')
}
```

**🎯 Призначення**: Забезпечити **безпечний вивід текстових полів**, замінюючи керуючі символи, нуль-байти та інші недрюковані символи на `.` — щоб уникнути "кракозябр" у логах та терміналі.

**🔢 Що вважається "графічним"?**
```
✅ Друковані: літери, цифри, пунктуація, пробіл, емодзі
❌ Недрюковані: 
   • Керуючі символи: \0, \n, \t, \r, \x01-\x1f
   • Спеціальні: \x7f (DEL), \x80-\x9f (C1 controls)
   • Невизначені: деякі Unicode code points
```

**🔄 Алгоритм:**
```
🔹 Вхід: r = '\n' (0x0a)
│
▼
🔹 unicode.IsGraphic('\n') → false
│
▼
🔹 Повертаємо '.' ✅
```

---

### 🔹 `EscapeUnprintables` — застосування до всього рядка

```go
func EscapeUnprintables(src string) string {
    // 🔹 strings.Map застосовує функцію до кожного rune у рядку
    return strings.Map(EscapeUnprintable, src)
}
```

**🎯 Призначення**: Очистити **весь рядок** від недрюкованих символів для безпечного виводу.

**🔢 Приклади:**
```
🔹 Вхід: "Hello\x00World"
🔹 Вихід: "Hello.World"  ← \0 замінено на '.'

🔹 Вхід: "Line1\nLine2\tTab"
🔹 Вихід: "Line1.Line2.Tab"  ← \n і \t замінено

🔹 Вхід: "Normal text"
🔹 Вихід: "Normal text"  ← без змін, всі символи друковані
```

**🎯 Навіщо це потрібно?**
- ✅ Запобігання "зламаному" виводу у терміналі (керуючі символи можуть змінювати колір, позицію курсора тощо)
- ✅ Безпека: уникнення ін'єкцій через спеціальні символи у логах
- ✅ Читабельність: `.` чітко вказує на наявність недрюкованого символу

### 🔹 Приклад 3: Валідація фіксовано-крапкових значень перед записом

```go
// 🔹 Функція для конвертації float у 16.16 fixed-point з валідацією
func floatToFixed1616(val float64) (int32, error) {
    // 🔹 Перевірка діапазону: ±32768 для цілої частини
    if val > 32768.0 || val < -32768.0 {
        return 0, fmt.Errorf("value %.2f out of range for 16.16 fixed-point", val)
    }
    
    // 🔹 Конвертація: val * 65536
    fixed := int32(val * (1 << 16))
    
    // 🔹 Перевірка на переповнення
    if float64(fixed)/(1<<16) != val && val != 0 {
        // 🔹 Попередження про втрату точності
        log.Printf("⚠️  Precision loss: %.6f → %d (16.16)", val, fixed)
    }
    
    return fixed, nil
}

// 🔹 Використання при створенні боксу:
func createTkhd(width, height float64) (*mp4.Tkhd, error) {
    widthFixed, err := floatToFixed1616(width)
    if err != nil {
        return nil, fmt.Errorf("invalid width: %w", err)
    }
    
    heightFixed, err := floatToFixed1616(height)
    if err != nil {
        return nil, fmt.Errorf("invalid height: %w", err)
    }
    
    return &mp4.Tkhd{
        Width:  widthFixed,
        Height: heightFixed,
        // ... інші поля ...
    }, nil
}
```

## ⚠️ Поширені помилки та як їх уникнути

| Помилка | Симптоми | Рішення |
|---------|----------|---------|
| Неправильне перетворення fixed-point | "1920.00000" замість "1920" для цілих значень | Перевіряйте `val&0xffff == 0` перед форматуванням |
| Втрата точності при конвертації float → fixed | `1.00001` стає `1.0` через округлення | Використовуйте достатню точність: 5 знаків для 16.16, 3 для 8.8 |
| Заміна корисних символів на `.` | "Line1\nLine2" → "Line1.Line2" (втрата структури) | Для логування використовуйте `EscapeUnprintables`, для збереження — ні |
| Ігнорування знаку у signed fixed-point | Від'ємні значення відображаються як додатні | Використовуйте `FormatSignedFixedFloat*` для signed, `Unsigned` для unsigned |
| Переповнення при конвертації | `float64(val)/(1<<16)` дає невірний результат для великих `val` | Перевіряйте діапазон вхідних значень перед конвертацією |

## 📋 Чекліст для вашого проекту

```
[ ] При форматуванні fixed-point чисел:
    • Використовуйте FormatSignedFixedFloat1616 для полів типу Width/Height/Matrix
    • Перевіряйте, чи значення ціле (val&0xffff==0) для кращої читабельності
    • Для 8.8 форматів використовуйте FormatSignedFixedFloat88 з 3 знаками після коми

[ ] При роботі з текстовими полями:
    • Використовуйте EscapeUnprintables для безпечного виводу у логах/терміналі
    • Не використовуйте очищення для збереження даних — тільки для відображення
    • Логувайте попередження, якщо знайдено недрюковані символи у важливих полях

[ ] Для валідації вхідних даних:
    • Перевіряйте діапазон float перед конвертацією у fixed-point
    • Попереджайте про втрату точності при округленні
    • Тестуйте крайні випадки: 0, ±цілі, ±дробові, максимальні/мінімальні значення

[ ] Для дебагу:
    • Логувайте сирі fixed-point значення у hex: log.Printf("🔢 Width raw: 0x%08x", val)
    • Порівнюйте форматований вивід з очікуваним: "1920" vs "1920.00000"
    • Тестуйте EscapeUnprintables з різними наборами символів: \0, \n, \t, Unicode

[ ] Для тестування:
    • Створюйте тестові випадки для fixed-point: цілі, дробові, від'ємні, граничні
    • Перевіряйте round-trip: float → fixed → formatted float → порівняння
    • Тестуйте EscapeUnprintables з контрольованими вхідними даними
```
