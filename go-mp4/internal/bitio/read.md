# 🔢 `bitio`: Низькорівневе бітове читання для парсингу MP4

Це **фундаментальний пакет** бібліотеки `go-mp4`, який реалізує **бітове читання** — можливість читати дані з потоку з точністю до окремого біта, що критично для парсингу форматів на кшталт MP4/ISOBMFF, де поля можуть мати довжину 1, 17, 29 або будь-яку іншу кількість біт.

---

## 🎯 Коротка відповідь

> **Це "бітовий скальпель" для MP4**: він дозволяє читати дані з точністю до окремого біта, буферизує байти для ефективності, підтримує вирівнювання та seek — все те, що потрібно для коректного парсингу бітових полів у стандартах ISO/IEC 14496.

---

## 🧱 Інтерфейси: Контракт бітового читання

### 🔹 `Reader` — базовий інтерфейс бітового читача

```go
type Reader interface {
    io.Reader  // 🔹 Стандартне читання байт

    // 🔹 Читання довільної кількості біт
    // 📐 alignment:
    //  |-1-byte-block-|--------------|--------------|
    //  |<-offset->|<-----------width---------------->|
    ReadBits(width uint) (data []byte, err error)

    // 🔹 Читання одного біта
    ReadBit() (bit bool, err error)
}
```

**🎯 Призначення**: Забезпечити єдиний API для:
- ✅ Читання байт (`io.Reader`) для сумісності
- ✅ Читання довільної кількості біт (`ReadBits`) для полів типу `size=17`
- ✅ Читання окремих біт (`ReadBit`) для прапорців

**🔢 Приклад використання:**
```
🔹 Поле: `Flag bool \`mp4:"0,size=1"\``
🔹 Виклик: bit, _ := reader.ReadBit()  // ← читаємо 1 біт

🔹 Поле: `Value uint16 \`mp4:"1,size=15"\``
🔹 Виклик: data, _ := reader.ReadBits(15)  // ← читаємо 15 біт
```

**🎯 Призначення**: Дозволити **переміщення по потоку** під час бітового читання — критично для:
- ✅ Повторного читання заголовків після парсингу дітей
- ✅ Пропуску невідомих боксів
- ✅ Відкату при помилках парсингу

## 🏗️ Реалізація: `reader` — бітовий буфер

### 🔹 Структура стану

```go
type reader struct {
    reader io.Reader  // 🔹 Базовий байтовий читач
    octet  byte       // 🔹 Поточний байт у буфері
    width  uint       // 🔹 Кількість непрочитаних біт у octet
}
```

**🎯 Призначення**: Інкапсулювати **стан бітового читання**:
- `octet`: байт, з якого читаємо біти по черзі
- `width`: скільки біт залишилось у цьому байті (8 → 0)

**🔢 Візуалізація стану:**
```
🔹 Початок: width=8, octet=0b10110010
📊 Біти:    [1][0][1][1][0][0][1][0]
            ↑
            читаємо звідси (зліва направо)

🔹 Після 3-х викликів ReadBit():
• width=5, прочитано: 1,0,1
• Наступний біт: 1 (четвертий зліва)
```

---

### 🔹 `NewReader` — конструктор

```go
func NewReader(r io.Reader) Reader {
    return &reader{reader: r}  // 🔹 width=0, octet=0 за замовчуванням
}
```

**🎯 Призначення**: Створити бітовий читач поверх будь-якого `io.Reader`.

### 🔹 `Read` — читання байт з перевіркою вирівнювання

```go
func (r *reader) Read(p []byte) (n int, err error) {
    // 🔹 Перевірка: чи вирівняні ми по байтах?
    if r.width != 0 {
        return 0, ErrInvalidAlignment  // ❌ Залишились непрочитані біти
    }
    return r.reader.Read(p)  // ✅ Делегуємо базовому читачу
}
```

**🎯 Призначення**: Дозволити читання байт **тільки коли бітовий буфер порожній** — запобігає змішуванню бітового та байтового читання.

**🔢 Приклад:**
```
🔹 Сценарій 1: width=0 → ✅ читаємо байти
🔹 Сценарій 2: width=3 → ❌ ErrInvalidAlignment
   • Чому? 3 непрочитані біти = неповний байт
   • Рішення: спочатку дочитати ці 3 біти через ReadBit/ReadBits
```

### 🔹 `ReadBit` — читання одного біта з буферизацією

```go
func (r *reader) ReadBit() (bool, error) {
    // 🔹 Крок 1: Якщо буфер порожній → читаємо новий байт
    if r.width == 0 {
        buf := make([]byte, 1)
        if n, err := r.reader.Read(buf); err != nil {
            return false, err
        } else if n != 1 {
            return false, ErrDiscouragedReader  // ❌ Неповне читання
        }
        r.octet = buf[0]   // 🔹 Зберігаємо байт у буфер
        r.width = 8        // 🔹 8 біт доступно для читання
    }

    // 🔹 Крок 2: Читаємо старший непрочитаний біт
    r.width--  // ← зменшуємо лічильник
    // 🔹 Маска: (octet >> width) & 0x01
    // Приклад: octet=0b10110010, width=7 → (0b10110010 >> 7) & 1 = 1
    return (r.octet>>r.width)&0x01 != 0, nil
}
```

**🔄 Алгоритм:**
```
🔹 Вхід: octet=0b10110010, width=8
│
▼
🔹 Виклик 1: ReadBit()
• width=8 → читаємо новий байт (вже є)
• width-- → width=7
• (0b10110010 >> 7) & 1 = 0b1 & 1 = 1 → ✅ true
│
▼
🔹 Виклик 2: ReadBit()
• width=7 → буфер не порожній
• width-- → width=6
• (0b10110010 >> 6) & 1 = 0b10 & 1 = 0 → ✅ false
│
▼
🔹 Виклик 3-8: аналогічно, читаємо решту біт
│
▼
🔹 Виклик 9:
• width=0 → буфер порожній → читаємо НОВИЙ байт
• Повторюємо цикл
```

**🎯 Ключова оптимізація**: Один системний виклик `Read()` на 8 біт, а не на кожен `ReadBit()`.

### 🔹 `ReadBits` — читання довільної кількості біт

```go
func (r *reader) ReadBits(size uint) ([]byte, error) {
    // 🔹 Крок 1: Розрахунок розміру результату у байтах
    bytes := (size + 7) / 8  // 🔹 Округлення вгору: 17 біт → 3 байти
    data := make([]byte, bytes)
    
    // 🔹 Крок 2: Обчислення офсету для вирівнювання вправо
    // 📐 Приклад: size=17, bytes=3 → offset = 24-17 = 7
    // • Це означає: перші 7 біт у 3-байтовому буфері будуть 0
    offset := (bytes * 8) - size
    
    // 🔹 Крок 3: Побітове заповнення результату
    for i := uint(0); i < size; i++ {
        bit, err := r.ReadBit()  // 🔹 Читаємо один біт
        if err != nil { return nil, err }
        
        // 🔹 Розрахунок позиції у байтовому масиві:
        byteIdx := (offset + i) / 8  // 🔹 Який байт?
        bitIdx := 7 - (offset+i)%8   // 🔹 Який біт у байті? (зліва направо)
        
        if bit {
            data[byteIdx] |= 0x1 << bitIdx  // 🔹 Встановлюємо біт
        }
    }
    
    return data, nil
}
```

**🔢 Приклад: читання 17 біт**
```
🔹 Вхід: size=17
🔹 bytes = (17+7)/8 = 3 байти
🔹 offset = 24 - 17 = 7

📊 Буфер data (3 байти = 24 біти):
[0][0][0][0][0][0][0] [b0][b1]...[b16]
 ↑                   ↑
 offset=7          останній біт

🔹 Цикл i=0..16:
• i=0: byteIdx=(7+0)/8=0, bitIdx=7-7%8=0 → data[0] |= 1<<0
• i=1: byteIdx=(7+1)/8=1, bitIdx=7-0=7 → data[1] |= 1<<7
• ...
• i=16: byteIdx=(7+16)/8=2, bitIdx=7-23%8=0 → data[2] |= 1<<0

✅ Результат: 17 біт вирівняні вправо у 3-байтовому масиві
```

**🎯 Призначення**: Забезпечити **право-вирівняне** представлення бітових полів — стандарт для цілих чисел у більшості форматів.

### 🔹 `Seek` — позиціонування з очищенням бітового буфера

```go
func (r *readSeeker) Seek(offset int64, whence int) (int64, error) {
    // 🔹 Перевірка вирівнювання при seek від поточної позиції
    if whence == io.SeekCurrent && r.reader.width != 0 {
        return 0, ErrInvalidAlignment  // ❌ Не можна seek'ати з середини байта
    }
    
    // 🔹 Виконуємо seek на базовому читачі
    n, err := r.seeker.Seek(offset, whence)
    if err != nil {
        return n, err
    }
    
    // 🔹 Ключовий крок: скидаємо бітовий буфер!
    r.reader.width = 0  // ← Наступне читання почнеться з нового байта
    
    return n, nil
}
```

**🎯 Призначення**: Забезпечити **коректне позиціонування** після бітового читання:
- ✅ Забороняє seek з середини байта (щоб уникнути втрати даних)
- ✅ Скидає буфер після успішного seek, щоб наступне читання почалося з нового байта

**🔢 Приклад:**
```
🔹 Сценарій 1: width=0 → ✅ Seek дозволено
• Seek(100, SeekStart) → позиція=100, width=0

🔹 Сценарій 2: width=3 → ❌ Seek від поточної позиції заборонено
• Чому? 3 непрочитані біти = неповний байт
• Рішення: спочатку дочитати біти або seek до абсолютної позиції

🔹 Сценарій 3: Seek(200, SeekStart) з width=5
• ✅ Дозволено (whence != SeekCurrent)
• Після seek: width=0 → буфер скинуто
```

```
   │   • Читаємо 8 біт другого байта

   │
   ▼

              ↑           ↑           ↑

```

### 🔹 Приклад 2: Ефективне читання великих масивів біт

```go
// 🔹 Функція для читання масиву прапорців (1 біт на елемент)
func readBitArray(r bitio.Reader, count uint) ([]bool, error) {
    result := make([]bool, count)
    
    for i := uint(0); i < count; i++ {
        bit, err := r.ReadBit()
        if err != nil {
            return nil, err
        }
        result[i] = bit
    }
    
    return result, nil
}

// 🔹 Оптимізована версія для великих масивів:
func readBitArrayOptimized(r bitio.Reader, count uint) ([]bool, error) {
    result := make([]bool, count)
    
    // 🔹 Читаємо байтами, де можливо
    bytesNeeded := (count + 7) / 8
    buf := make([]byte, bytesNeeded)
    
    // 🔹 Спроба читання байт (якщо вирівняно)
    if n, err := r.Read(buf); err == nil && n == len(buf) {
        // 🔹 Розпаковуємо біти з байт
        for i := uint(0); i < count; i++ {
            byteIdx := i / 8
            bitIdx := 7 - i%8
            result[i] = (buf[byteIdx]>>bitIdx)&0x01 != 0
        }
        return result, nil
    }
    
    // 🔹 Fallback на побітове читання
    return readBitArray(r, count)
}
```

## ⚠️ Поширені помилки та як їх уникнути

| Помилка | Симптоми | Рішення |
|---------|----------|---------|
| Читання байт при `width != 0` | `ErrInvalidAlignment` | Завжди перевіряйте вирівнювання перед `Read()`: дочитайте залишок біт |
| Seek з середини байта | `ErrInvalidAlignment` | Використовуйте `SeekStart`/`SeekEnd` замість `SeekCurrent`, або спочатку вирівняйтеся |
| Неправильне вирівнювання у `ReadBits` | Зміщені дані, неправильні значення | Пам'ятайте: `ReadBits` вирівнює вправо — останні біти у останньому байті |
| Ігнорування `ErrDiscouragedReader` | Повільне читання, зависання | Використовуйте буферизовані читачі: `bufio.NewReader()` |
| Забути скинути буфер після seek | Читання старих даних після позиціонування | `readSeeker.Seek` автоматично скидає `width=0` — не робіть цього вручну |

## 📋 Чекліст для вашого проекту

```
[ ] При роботі з бітовими полями:
    • Переконайтеся, що сума розмірів полів кратна 8 для вирівнювання
    • Використовуйте `ReadBits` замість множинних `ReadBit` для ефективності
    • Перевіряйте `width == 0` перед викликом `Read()` для байтового читання

[ ] Для позиціонування:
    • Уникайте `SeekCurrent` якщо можливі непрочитані біти
    • Після `Seek` завжди припускайте, що буфер скинуто (width=0)
    • Для відкату: зберігайте позицію через `Seek(0, SeekCurrent)` перед читанням

[ ] Для оптимізації:
    • Групуйте бітові поля у байти для зменшення викликів `ReadBit`
    • Використовуйте `bufseekio.NewReadSeeker` для буферизації з підтримкою seek
    • Для великих масивів: читайте байтами та розпаковуйте біти вручну

[ ] Для дебагу:
    • Логувайте стан: log.Printf("🔢 width=%d, octet=0x%02x", r.width, r.octet)
    • Перевіряйте вирівнювання після операцій: if width%8 != 0 { ... }
    • Тестуйте з відомими бітовими послідовностями для валідації логіки

[ ] Для тестування:
    • Створюйте тестові потоки з контрольованими бітовими даними
    • Перевіряйте round-trip: запис біт → читання → порівняння
    • Тестуйте крайні випадки: 1 біт, 17 біт, 64 біти, непарні розміри
```
