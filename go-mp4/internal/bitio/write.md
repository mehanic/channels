# ✍️ `bitio/writer.go`: Низькорівневе бітове записування для парсингу MP4

Це **фундаментальний пакет** бібліотеки `go-mp4`, який реалізує **бітове записування** — можливість записувати дані у потік з точністю до окремого біта, що критично для створення форматів на кшталт MP4/ISOBMFF, де поля можуть мати довжину 1, 17, 29 або будь-яку іншу кількість біт.

---

## 🎯 Коротка відповідь

> **Це "бітове перо" для MP4**: він дозволяє записувати дані з точністю до окремого біта, буферизує байти для ефективності, підтримує вирівнювання та автоматичне скидання — все те, що потрібно для коректного створення бітових полів у стандартах ISO/IEC 14496.

---

## 🧱 Інтерфейс: Контракт бітового запису

### 🔹 `Writer` — базовий інтерфейс бітового записувача

```go
type Writer interface {
    io.Writer  // 🔹 Стандартний запис байт

    // 🔹 Запис довільної кількості біт
    // 📐 alignment:
    //  |-1-byte-block-|--------------|--------------|
    //  |<-offset->|<-----------width---------------->|
    WriteBits(data []byte, width uint) error

    // 🔹 Запис одного біта
    WriteBit(bit bool) error
}
```

**🎯 Призначення**: Забезпечити єдиний API для:
- ✅ Запису байт (`io.Writer`) для сумісності
- ✅ Запису довільної кількості біт (`WriteBits`) для полів типу `size=17`
- ✅ Запису окремих біт (`WriteBit`) для прапорців

**🔢 Приклад використання:**
```
🔹 Поле: `Flag bool \`mp4:"0,size=1"\``
🔹 Виклик: writer.WriteBit(true)  // ← записуємо 1 біт

🔹 Поле: `Value uint16 \`mp4:"1,size=15"\``
🔹 Виклик: writer.WriteBits([]byte{0x01, 0x63}, 15)  // ← записуємо 15 біт
```

## 🏗️ Реалізація: `writer` — бітовий буфер

### 🔹 Структура стану

```go
type writer struct {
    writer io.Writer  // 🔹 Базовий байтовий записувач
    octet  byte       // 🔹 Поточний байт у буфері
    width  uint       // 🔹 Кількість записаних біт у octet
}
```

**🎯 Призначення**: Інкапсулювати **стан бітового запису**:
- `octet`: байт, у який записуємо біти по черзі
- `width`: скільки біт вже записано у цьому байті (0 → 8)

**🔢 Візуалізація стану:**
```
🔹 Початок: width=0, octet=0b00000000
📊 Біти:    [ ][ ][ ][ ][ ][ ][ ][ ]
            ↑
            записуємо сюди (зліва направо)

🔹 Після 3-х викликів WriteBit(true, false, true):
• width=3, octet=0b10100000
• Наступний біт запишеться на позицію 3 (зліва)
```

### 🔹 `NewWriter` — конструктор

```go
func NewWriter(w io.Writer) Writer {
    return &writer{writer: w}  // 🔹 width=0, octet=0 за замовчуванням
}
```

**🎯 Призначення**: Створити бітовий записувач поверх будь-якого `io.Writer`.

### 🔹 `Write` — запис байт з перевіркою вирівнювання

```go
func (w *writer) Write(p []byte) (n int, err error) {
    // 🔹 Перевірка: чи вирівняні ми по байтах?
    if w.width != 0 {
        return 0, ErrInvalidAlignment  // ❌ Є не записані біти у буфері
    }
    return w.writer.Write(p)  // ✅ Делегуємо базовому записувачу
}
```

**🎯 Призначення**: Дозволити запис байт **тільки коли бітовий буфер порожній** — запобігає змішуванню бітового та байтового запису.

**🔢 Приклад:**
```
🔹 Сценарій 1: width=0 → ✅ записуємо байти
🔹 Сценарій 2: width=3 → ❌ ErrInvalidAlignment
   • Чому? 3 записані біти = неповний байт
   • Рішення: спочатку записати ще 5 біт через WriteBit/WriteBits, щоб завершити байт
```

### 🔹 `WriteBit` — запис одного біта з буферизацією

```go
func (w *writer) WriteBit(bit bool) error {
    // 🔹 Крок 1: Встановлюємо біт у поточний октет
    if bit {
        // 🔹 Маска: 0x1 << (7 - width)
        // Приклад: width=0 → 0x1 << 7 = 0b10000000 (старший біт)
        //          width=1 → 0x1 << 6 = 0b01000000
        w.octet |= 0x1 << (7 - w.width)
    }
    // 🔹 Якщо bit=false, просто залишаємо 0 на цій позиції
    
    // 🔹 Крок 2: Збільшуємо лічильник записаних біт
    w.width++
    
    // 🔹 Крок 3: Якщо байт заповнений → записуємо у потік
    if w.width == 8 {
        if _, err := w.writer.Write([]byte{w.octet}); err != nil {
            return err
        }
        // 🔹 Скидаємо буфер для наступного байта
        w.octet = 0x00
        w.width = 0
    }
    return nil
}
```

**🔄 Алгоритм:**
```
🔹 Вхід: octet=0b00000000, width=0
│
▼
🔹 Виклик 1: WriteBit(true)
• w.octet |= 0x1 << (7-0) = 0x1 << 7 = 0b10000000
• octet=0b10000000, width=1
│
▼
🔹 Виклик 2: WriteBit(false)
• bit=false → нічого не змінюємо
• octet=0b10000000, width=2
│
▼
🔹 Виклик 3: WriteBit(true)
• w.octet |= 0x1 << (7-2) = 0x1 << 5 = 0b00100000
• octet=0b10100000, width=3
│
▼
🔹 Виклики 4-8: заповнюємо решту біт
│
▼
🔹 Виклик 8: width=8 → записуємо байт у потік
• writer.Write([]byte{0b10100000})
• Скидаємо: octet=0x00, width=0
```

**🎯 Ключова оптимізація**: Один системний виклик `Write()` на 8 біт, а не на кожен `WriteBit()`.

### 🔹 `WriteBits` — запис довільної кількості біт

```go
func (w *writer) WriteBits(data []byte, width uint) error {
    // 🔹 Крок 1: Розрахунок загальної кількості біт у вхідних даних
    length := uint(len(data)) * 8  // 🔹 Напр. 2 байти = 16 біт
    
    // 🔹 Крок 2: Обчислення офсету для вирівнювання вправо
    // 📐 Приклад: width=17, length=24 (3 байти) → offset = 24-17 = 7
    // • Це означає: перші 7 біт у 3-байтовому масиві ігноруються
    offset := length - width
    
    // 🔹 Крок 3: Побітове записування з вхідного масиву
    for i := offset; i < length; i++ {
        // 🔹 Розрахунок позиції у байтовому масиві:
        oi := i / 8  // 🔹 Який байт?
        
        // 🔹 Витягуємо потрібний біт з байта:
        // (data[oi] >> (7 - i%8)) & 0x01
        // Приклад: i=7, oi=0 → (data[0] >> 0) & 1 = молодший біт
        bit := (data[oi] >> (7 - i%8)) & 0x01 != 0
        
        // 🔹 Записуємо біт через WriteBit (з буферизацією)
        if err := w.WriteBit(bit); err != nil {
            return err
        }
    }
    return nil
}
```

**🔢 Приклад: запис 17 біт**
```
🔹 Вхід: data=[0x01, 0x63, 0xd5], width=17
🔹 length = 3*8 = 24 біт
🔹 offset = 24 - 17 = 7

📊 Вхідний масив (3 байти = 24 біти):
[00000001] [01100011] [11010101]
 ↑                   ↑
offset=7           останній біт

🔹 Цикл i=7..23:
• i=7: oi=7/8=0, біт = (0x01 >> (7-7%8)) & 1 = (0x01 >> 0) & 1 = 1
• i=8: oi=8/8=1, біт = (0x63 >> (7-0)) & 1 = (0x63 >> 7) & 1 = 0
• ...
• i=23: останній біт 0xd5

✅ Результат: 17 біт записані по черзі через WriteBit, з автоматичним скиданням байт
```

**🎯 Призначення**: Забезпечити **право-вирівняне** записування бітових полів — стандарт для цілих чисел у більшості форматів.

## 🔄 Потік даних: Повний приклад запису

```
🔹 Вихід: потік байт [0xb5, 0x63, 0xd5, ...]
│
▼
🔹 writer.WriteBits([]byte{0x5a}, 7):
   │
   ├── 🔹 length=8, offset=8-7=1
   │
   ├── 🔹 Цикл i=1..7:
   │   • i=1: oi=0, біт = (0x5a >> (7-1%8)) & 1 = (0x5a >> 6) & 1 = 0
   │   • WriteBit(0) → octet=0b00000000, width=1
   │   • i=2: біт = (0x5a >> 5) & 1 = 1 → WriteBit(1) → octet=0b01000000, width=2
   │   • ...
   │   • i=7: біт = (0x5a >> 0) & 1 = 0 → WriteBit(0) → width=7
   │
   ▼
🔹 Після циклу: width=7, octet=0b01011010 (але ще не записано!)
│
▼
🔹 Наступний WriteBit(true):
   • Записуємо біт → width=8
   • width==8 → записуємо байт 0b01011010 = 0x5a у потік ✅
   • Скидаємо: octet=0, width=0
```

## 🛠️ Практичне використання у вашому CCTV HLS Processor

### 🔹 Приклад 1: Запис кастомного боксу з бітовими полями

```go
// 🔹 Структура з бітовими полями
type CustomFlags struct {
    mp4.Box
    Flag1    bool  `mp4:"0,size=1"`   // 🔹 1 біт
    Flag2    bool  `mp4:"1,size=1"`   // 🔹 1 біт
    Reserved uint8 `mp4:"2,size=4"`   // 🔹 4 біти
    Value    uint8 `mp4:"3,size=2"`   // 🔹 2 біти
    // ✅ Разом: 1+1+4+2 = 8 біт = 1 байт
}

// 🔹 Ручний запис через bitio (якщо автоматичний не підходить)
func writeCustomFlags(w bitio.Writer, cf *CustomFlags) error {
    // 🔹 Запис біт по черзі
    if err := w.WriteBit(cf.Flag1); err != nil { return err }
    if err := w.WriteBit(cf.Flag2); err != nil { return err }
    
    // 🔹 Запис 4-бітного паддінгу (завжди 0)
    for i := 0; i < 4; i++ {
        if err := w.WriteBit(false); err != nil { return err }
    }
    
    // 🔹 Запис 2-бітного значення (старші біти перші)
    if err := w.WriteBit((cf.Value & 0x02) != 0); err != nil { return err }  // старший біт
    if err := w.WriteBit((cf.Value & 0x01) != 0); err != nil { return err }  // молодший біт
    
    // 🔹 Перевірка вирівнювання (опціонально)
    // У реальності це робиться на рівні marshal
    
    return nil
}
```

### 🔹 Приклад 2: Ефективний запис великих масивів біт

```go
// 🔹 Функція для запису масиву прапорців (1 біт на елемент)
func writeBitFlags(w bitio.Writer, flags []bool) error {
    for _, flag := range flags {
        if err := w.WriteBit(flag); err != nil {
            return err
        }
    }
    return nil
}

// 🔹 Оптимізована версія для великих масивів:
func writeBitFlagsOptimized(w bitio.Writer, flags []bool) error {
    // 🔹 Групуємо біти у байти для зменшення викликів
    bytesNeeded := (len(flags) + 7) / 8
    buf := make([]byte, bytesNeeded)
    
    // 🔹 Упаковуємо біти у байти
    for i, flag := range flags {
        if flag {
            byteIdx := i / 8
            bitIdx := 7 - i%8  // 🔹 Біти зліва направо
            buf[byteIdx] |= 0x1 << bitIdx
        }
    }
    
    // 🔹 Записуємо повні байти через Write (якщо вирівняно)
    // 🔹 Залишок біт записуємо через WriteBits
    fullBytes := len(flags) / 8
    if fullBytes > 0 {
        if _, err := w.Write(buf[:fullBytes]); err != nil {
            return err
        }
    }
    
    // 🔹 Запис залишку біт (якщо є)
    remainder := len(flags) % 8
    if remainder > 0 {
        lastByte := buf[fullBytes]
        if err := w.WriteBits([]byte{lastByte}, uint(remainder)); err != nil {
            return err
        }
    }
    
    return nil
}
```

## ⚠️ Поширені помилки та як їх уникнути

| Помилка | Симптоми | Рішення |
|---------|----------|---------|
| Запис байт при `width != 0` | `ErrInvalidAlignment` | Завжди перевіряйте вирівнювання перед `Write()`: завершіть поточний байт через `WriteBit` |
| Неправильне вирівнювання у `WriteBits` | Зміщені дані, неправильні значення | Пам'ятайте: `WriteBits` очікує право-вирівняні дані — старші біти у старших позиціях байта |
| Забути завершити останній байт | Неповні дані у файлі, помилки парсингу | Завжди вирівнюйте до 8 біт в кінці боксу: записуйте паддінг-біти (0) |
| Неправильний порядок біт у `WriteBits` | Дані записані у зворотному порядку | Перевірте логіку: `(data[oi] >> (7 - i%8)) & 0x01` — біти зліва направо |
| Ігнорування помилок `WriteBit` | Пошкоджені файли без повідомлення про помилку | Завжди перевіряйте `err` після кожного `WriteBit`/`WriteBits` |

## 📋 Чекліст для вашого проекту

```
[ ] При записі бітових полів:
    • Переконайтеся, що сума розмірів полів кратна 8 для вирівнювання
    • Використовуйте `WriteBits` замість множинних `WriteBit` для ефективності
    • Перевіряйте `width == 0` перед викликом `Write()` для байтового запису

[ ] Для вирівнювання:
    • Завжди завершуйте бокс вирівнюванням до 8 біт (паддінг-біти = 0)
    • Перевіряйте, що `width == 0` після завершення запису боксу
    • Для вкладених структур: вирівнюйте кожен бокс окремо

[ ] Для оптимізації:
    • Групуйте бітові поля у байти для зменшення викликів `WriteBit`
    • Використовуйте буферизовані записувачі: `bufio.NewWriter()`
    • Для великих масивів: упаковуйте біти у байти вручну перед записом

[ ] Для дебагу:
    • Логувайте стан: log.Printf("🔢 width=%d, octet=0x%02x", w.width, w.octet)
    • Перевіряйте вирівнювання після операцій: if width%8 != 0 { ... }
    • Тестуйте з відомими бітовими послідовностями для валідації логіки

[ ] Для тестування:
    • Створюйте тестові буфери з контрольованими бітовими даними
    • Перевіряйте round-trip: запис біт → читання → порівняння
    • Тестуйте крайні випадки: 1 біт, 17 біт, 64 біти, непарні розміри
```
