# 🧪 Тест `TestBoxTypesISO14496_30`: WebVTT субтитри у MP4

Це **юніт-тест** для бібліотеки `go-mp4`, який перевіряє коректну роботу **серіалізації/десеріалізації** боксів стандарту **ISO/IEC 14496-30** (WebVTT у ISOBMFF).

---

## 🎯 Коротка відповідь

> **Цей тест гарантує, що текстові субтитри у форматі WebVTT коректно вбудовуються у MP4/fMP4-файли** — критично для підтримки багатомовних субтитрів, описів для слабозорих та метаданих у вашому HLS-стрімі.

## 🔍 Детальний розбір тест-кейсів

### 🔹 Конфігураційні бокси

#### ✅ `vttC` — WebVTTConfigurationBox
```go
{
    name: "vttC",
    src: &WebVTTConfigurationBox{Config: "WEBVTT\n"},
    bin: []byte{'W', 'E', 'B', 'V', 'T', 'T', '\n'},  // просто байти рядка
    str: `Config="WEBVTT."`,  // Stringify замінює \n на . для читабельності
},
```
**🎯 Призначення**: Глобальні налаштування (CSS, регіони) для всієї доріжки субтитрів.

---

#### ✅ `vlab` — WebVTTSourceLabelBox
```go
{
    name: "vlab",
    src: &WebVTTSourceLabelBox{SourceLabel: "Source"},
    bin: []byte{'S', 'o', 'u', 'r', 'c', 'e'},
    str: `SourceLabel="Source"`,
},
```
**🎯 Призначення**: Мітка джерела ("українська", "англійська", "аудіо-опис").

---

#### ✅ `wvtt` — WVTTSampleEntry
```go
{
    name: "wvtt",
    src: &WVTTSampleEntry{
        SampleEntry: SampleEntry{
            DataReferenceIndex: 0x1234,  // 4660 десяткове
        },
    },
    bin: []byte{0x0,0x0,0x0,0x0,0x0,0x0, 0x12,0x34},  // 6 байт reserved + 2 байти index
    str: `DataReferenceIndex=4660`,
},
```
**🎯 Призначення**: Оголошення у `stsd`: "ця доріжка містить WebVTT".

**🎯 Призначення**: Числовий ID джерела для швидкої фільтрації за мовою.

---

#### ✅ `ctim` — CueTimeBox ⭐ Найважливіший для синхронізації!
```go
{
    name: "ctim",
    src: &CueTimeBox{CueCurrentTime: "00:00:00.000"},

    str: `CueCurrentTime="00:00:00.000"`,
},
```
**🎯 Призначення**: Час появи к'ю у форматі `"HH:MM:SS.mmm"`. Плеєр парсить цей рядок для синхронізації з відео.

---

#### ✅ `iden` — CueIDBox

**🎯 Призначення**: Унікальний ідентифікатор к'ю для посилань/аналітики.

```go
{

```
**🎯 Призначення**: Налаштування відображення: позиція, вирівнювання, розмір.

#### ✅ `payl` — CuePayloadBox 🔥 Основний текст субтитрів!

## 🔄 Чотири операції, що тестуються

### 🔹 1. `Marshal` — серіалізація (структура → байти)
```go
buf := bytes.NewBuffer(nil)
n, err := Marshal(buf, tc.src, tc.ctx)
assert.Equal(t, tc.bin, buf.Bytes())  // байт в байт!
```
**Перевіряє**: Чи коректно перетворюється рядок у байти без втрат.

---

### 🔹 2. `Unmarshal` — десеріалізація (байти → структура)
```go
r := bytes.NewReader(tc.bin)
n, err = Unmarshal(r, uint64(len(tc.bin)), tc.dst, tc.ctx)
assert.Equal(t, tc.src, tc.dst)  // 🔁 round-trip: ідемпотентність!
```
**Перевіряє**: Чи відновлюється структура точно такою ж після читання.

---

### 🔹 3. `UnmarshalAny` — динамічний парсинг за типом
```go
dst, n, err := UnmarshalAny(bytes.NewReader(tc.bin), tc.src.GetType(), ...)
assert.Equal(t, tc.src, dst)
```
**Перевіряє**: Чи працює універсальний парсер для невідомих наперед типів боксів.

---

### 🔹 4. `Stringify` — людино-читабельний вивід
```go
str, err := Stringify(tc.src, tc.ctx)
assert.Equal(t, tc.str, str)
```
**Перевіряє**: Чи коректно формується рядок для дебагу/логів.

> 💡 **Зверніть увагу**: У `str` символ `\n` замінено на `.` — це спеціальна обробка для читабельності логів.

## 🎯 Чому цей тест критичний для вашого HLS-процесора?

### 🔹 Сценарій 1: Синхронізація субтитрів з відео

```
📡 Ви додаєте українські субтитри до стріму:
1. Формуєте к'ю: {Time: "00:01:23.456", Text: "Привіт!"}
2. Marshal перетворює у байти: '0','0',':',...,'П','р','и','в','і','т','!'
3. Записуєте у fMP4-сегмент
4. Плеєр читає: парсить "00:01:23.456" → показує текст у потрібний момент

❌ Без тесту: помилка в кодуванні часу → субтитри з'являються на 2 секунди пізніше
```

### 🔹 Сценарій 2: Багатомовність

```
🌐 Ви підтримуєте 3 мови субтитрів:
1. Для кожної мови: окремий `vlab` з міткой ("uk", "en", "ru")
2. Клієнт обирає мову → фільтрує к'ю за `SourceLabel`
3. Відображає тільки обрану мову

✅ Тест гарантує, що мітки коректно серіалізуються/десеріалізуються
❌ Без тесту: "uk" читається як "u" → фільтрація ламається
```

### 🔹 Сценарій 3: Доступність для слабозорих

```
♿ Ви додаєте аудіо-опис:
1. Створюєте к'ю з `vtta`: `"[шум вітру]"`, `"[аплодисменти]"`
2. Додаєте `sttg` з `region:lower-third` щоб не перекривати основні субтитри
3. Плеєр з підтримкою accessibility показує опис

✅ Тест гарантує, що додатковий текст не губиться при серіалізації
❌ Без тесту: `"[шум вітру]"` обрізається → слабозорі не чують контекст
```

## ⚠️ Поширені помилки та як їх уникнути

| Помилка | Симптоми | Рішення |
|---------|----------|---------|
| Неправильний формат часу в `ctim` | Субтитри не синхронізовані | Завжди використовуйте `"HH:MM:SS.mmm"` з 3 цифрами мілісекунд |
| Відсутній `payl` у `vttc` | К'ю не відображається | Завжди додавайте `CuePayloadBox` з текстом |
| Неправильне кодування тексту | Кирилиця/емодзі → "кракозябри" | Використовуйте UTF-8 без BOM, перевіряйте кодування |
| Переповнення `sttg` | Плеєр ігнорує всі налаштування | Використовуйте тільки підтримувані параметри: `align`, `line`, `position`, `size` |
| Забути `vttC` у конфігурації | Глобальні стилі не працюють | Додайте `WebVTTConfigurationBox` при створенні доріжки |

## 📋 Чекліст для вашого проекту

```
[ ] При додаванні субтитрів:
    • Створіть текстову доріжку з `wvtt` Sample Entry у `stsd`
    • Додайте `vttC` бокс з базовими стилями (напр. `::cue { font-size: 18px; }`)
    • Для кожного к'ю: `vttc` → `ctim` + `payl` (+ опціонально `sttg`, `iden`)

[ ] Для багатомовності:
    • Використовуйте `vlab` для міток: "uk", "en", "ru"
    • Генеруйте окремий HLS-плейлист для кожної мови
    • Додавайте `#EXT-X-MEDIA:TYPE=SUBTITLES` у головний .m3u8

[ ] Для доступності:
    • Додавайте `vtta` з описом звуків: `"[шум вітру]"`, `"[музика]"`
    • Використовуйте `sttg` з `region:lower-third` для не-перекриваючих описів
    • Тестуйте на плеєрах з підтримкою accessibility

[ ] Для дебагу:
    • Логуйте текст к'ю: log.Printf("📝 Cue: %q", cueText)
    • Перевіряйте формат часу: if !isValidWebVTTTime(ctim) { ... }
    • Використовуйте `Stringify()` для виводу структури: log.Printf("📦 %s", Stringify(vttc, ctx))

[ ] Для тестування:
    • Перевіряйте синхронізацію: субтитри мають з'являтися вчасно (+/- 50мс)
    • Тестуйте з різними кодуваннями: кирилиця, арабська, емодзі
    • Перевіряйте відтворення у різних плеєрах: VLC, hls.js, Safari, ExoPlayer
```
