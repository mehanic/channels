# 🧪 Тест `TestBoxTypes3GPP`: Повний розбір

Це **інтеграційний тест** для бібліотеки `go-mp4`, який перевіряє коректну роботу **серіалізації/десеріалізації** 3GPP-рядків (метадані типу `titl`, `dscp` тощо).

---

## 🎯 Коротка відповідь

> **Цей тест гарантує, що метадані (назва, опис, мова) коректно перетворюються між структурою Go та бінарним форматом MP4** — і назад.

## 📋 Структура тесту

```go
func TestBoxTypes3GPP(t *testing.T) {
    // 1. Масив тест-кейсів
    testCases := []struct {
        name string           // Назва тесту
        src  IImmutableBox    // Вихідна структура (для запису)
        dst  IBox             // Порожня структура (для читання)
        bin  []byte           // Очікувані байти у файлі
        str  string           // Очікуваний рядок для дебагу (Stringify)
        ctx  Context          // Контекст парсингу
    }{ ... }

    // 2. Запуск кожного кейсу
    for _, tc := range testCases {
        t.Run(tc.name, func(t *testing.T) {
            // Тестова логіка: Marshal → Unmarshal → UnmarshalAny → Stringify
        })
    }
}
```

> 🎯 **Pattern**: Один тест-кейс перевіряє **чотири операції** одночасно — це економить час і гарантує консистентність.

## 🔍 Детальний розбір тест-кейсу: "udta 3gpp string"

### ✅ Вхідні дані (`src`)

```go
src: &Udta3GppString{
    AnyTypeBox: AnyTypeBox{Type: StrToBoxType("titl")},  // тип боксу: "titl"
    Language:   [3]byte{0x5, 0xe, 0x7},                   // мова: "eng" в ISO-639-2/T
    Data:       []byte("SING"),                           // текст: "SING"
},
```

**Розбір `Language: [3]byte{0x5, 0xe, 0x7}`**:
```
🔤 ISO-639-2/T кодування: 'a'=1, 'b'=2, ..., 'z'=26

   0x05 = 5  → 5-та літера = 'e'
   0x0e = 14 → 14-та літера = 'n'  
   0x07 = 7  → 7-ма літера = 'g'

✅ Разом: "eng" (англійська)
```

> 🎯 **Важливо**: Це **не** ASCII-байти! Це 5-бітне кодування, де кожна літера займає 5 біт.

---

### ✅ Очікувані байти (`bin`)

```go
bin: []byte{
    0,                // version = 0 (FullBox)
    0x00, 0x00, 0x00, // flags = 0x000000 (FullBox)
    0x15, 0xc7,       // language: 0x5,0xe,0x7 → упаковано в 2 байти
    0x53, 0x49, 0x4e, 0x47, // data: "SING" в ASCII/UTF-8
},
```

**🔢 Бітова упаковка `Language`**:
```
📐 Вхід: [3]byte{0x05, 0x0e, 0x07} = 5 біт + 5 біт + 5 біт = 15 біт

🔤 Бінарно:
   0x05 = 00101
   0x0e = 01110
   0x07 = 00111

📦 Разом (15 біт): 00101 01110 00111

🔢 Групуємо по 8 біт (байтах):
   Байт 0: 00101011 = 0x2B? Ні, чекайте...
   
🔄 Правильне вирівнювання (з урахуванням 1 біта Pad):
   [Pad:0][00101][01110][00111] = 16 біт = 2 байти
   
   Байт 0: 0[0010101] = 0001 0101 = 0x15 ✅
   Байт 1: [11000111] = 1100 0111 = 0xC7 ✅

🎯 Результат: [0x15, 0xC7] — саме те, що в тесті!
```

---

### ✅ Очікуваний рядок для дебагу (`str`)

```go
str: `Version=0 Flags=0x000000 Language="eng" Data="SING"`,
```

**Навіщо це?** Метод `Stringify()` використовується для:
- 🪲 Дебагу парсингу (логи)
- 🧪 Порівняння результатів у тестах
- 📋 Генерації документації

---

### ✅ Контекст (`ctx`)

```go
ctx: Context{UnderUdta: true},
```

**Чому це важливо?** 
- Бокс `titl` може зустрічатися **різних місцях** файлу
- Тільки всередині `udta` він має структуру `Udta3GppString`
- Без `UnderUdta=true` парсер може неправильно інтерпретувати дані

## 🔄 Чотири операції, що тестуються

### 🔹 1. `Marshal` — серіалізація (структура → байти)

```go
buf := bytes.NewBuffer(nil)
n, err := Marshal(buf, tc.src, tc.ctx)
require.NoError(t, err)
assert.Equal(t, uint64(len(tc.bin)), n)  // перевірка розміру
assert.Equal(t, tc.bin, buf.Bytes())      // перевірка вмісту
```

**Що перевіряємо:**
```
📦 Вхід: Udta3GppString{Language:[3]byte{5,14,7}, Data:"SING"}
              │
              ▼
         Marshal() + Context{UnderUdta:true}
              │
              ▼
📤 Вихід: []byte{0, 0,0,0, 0x15,0xc7, 'S','I','N','G'}
              │
              ▼
✅ Порівнюємо з очікуваним масивом tc.bin
```

---

### 🔹 2. `Unmarshal` — десеріалізація (байти → структура)

```go
r := bytes.NewReader(tc.bin)
n, err = Unmarshal(r, uint64(len(tc.bin)), tc.dst, tc.ctx)
require.NoError(t, err)
assert.Equal(t, uint64(buf.Len()), n)  // прочитано стільки ж, скільки записано
assert.Equal(t, tc.src, tc.dst)         // структура відновлена точно!
```

**Ключова перевірка**: `assert.Equal(t, tc.src, tc.dst)`
```
🔁 Round-trip тест:
   Структура → байти → Структура'
   
   Якщо Структура == Структура' → ✅ серіалізація ідемпотентна
   Якщо ні → ❌ десь втрата даних або помилка вирівнювання
```

---

### 🔹 3. `UnmarshalAny` — динамічний парсинг за типом

```go
dst, n, err := UnmarshalAny(
    bytes.NewReader(tc.bin), 
    tc.src.GetType(),        // "titl"
    uint64(len(tc.bin)), 
    tc.ctx,
)
require.NoError(t, err)
assert.Equal(t, tc.src, dst)  // те саме, що й при прямому Unmarshal
```

**Навіщо `UnmarshalAny`?**
```
🔍 Сценарій: Ви читаєте файл і не знаєте наперед тип боксу

1. ReadBoxInfo() → дізнаєтесь: тип="titl", size=12
2. UnmarshalAny(r, BoxType("titl"), size, ctx) 
3. Бібліотека сама:
   • Знаходить зареєстровану структуру для "titl" (Udta3GppString)
   • Створює її екземпляр
   • Парсить дані
   • Повертає готовий об'єкт

✅ Ви не пишете switch/case для 100+ типів боксів!
```

---

### 🔹 4. `Stringify` — людський формат для дебагу

```go
str, err := Stringify(tc.src, tc.ctx)
require.NoError(t, err)
assert.Equal(t, tc.str, str)  // перевірка формату виводу
```

**Приклад виводу `Stringify()`**:
```go
// Для нашого кейсу:
`Version=0 Flags=0x000000 Language="eng" Data="SING"`

// Для більш складного боксу:
`Version=1 Flags=0x000001 TrackID=123 BaseDataOffset=0x1000 
  SampleCount=42 Samples=[{Duration:1024, Size:4096}, ...]`
```

> 🎯 **Перевага**: Замість дампів байтів (`[00 15 c7 53 49...]`) ви бачите зрозумілі значення.

## 🛠️ Додаткові перевірки: позиція читача

```go
// Після Unmarshal: перевіряємо, що курсор в кінці прочитаних даних
s, err := r.Seek(0, io.SeekCurrent)
require.NoError(t, err)
assert.Equal(t, int64(buf.Len()), s)  // прочитано рівно n байт
```

**Навіщо?** Щоб уникнути "з'їдання" наступного боксу:
```
📦 Файл: [moof][traf][trun][mdat]
              │
              ▼
        Парсимо "traf"
              │
              ▼
❌ Якщо Unmarshal прочитає зайві 2 байти → "trun" пошкоджений
✅ Якщо курсор точно в кінці "traf" → "trun" читається коректно
```

## 🎯 Чому цей тест критичний для вашого HLS-процесора?

### 🔹 Сценарій 1: Коректне відображення метаданих у плеєрі

```
📡 Ви додаєте назву передачі до сегмента:
1. Marshal перетворює "Ранкові новини" → бінарний формат
2. Плеєр (VLC, hls.js) читає бокс "titl"
3. Декодує мову "ukr" → показує назву з правильним шрифтом

❌ Без тесту: помилка в кодуванні мови → плеєр показує "und" (undefined)
```

### 🔹 Сценарій 2: Ідемпотентність при модифікації файлів

```
📝 Ви читаєте сегмент → змінюєте метадані → записуєте назад:
1. Unmarshal: байти → структура
2. Змінюєте meta.Data = []byte("Нова назва")
3. Marshal: структура → нові байти

✅ Тест гарантує, що кроки 1 і 3 — взаємно обернені
❌ Без тесту: після перезапису файл може стати нечитабельним
```

### 🔹 Сценарій 3: Динамічний парсинг невідомих боксів

```
🔍 Камера надсилає сегмент з новим типом метаданих:
• Ви не очікували бокс "airt" (час ефіру)
• Але він зареєстрований через AddAnyTypeBoxDefEx

✅ UnmarshalAny автоматично використає Udta3GppString для "airt"
❌ Без реєстрації/тесту: бокс ігнорується або викликає помилку
```

## 🧪 Як додати свій тест-кейс?

```go
// Додайте у масив testCases новий елемент:
{
    name: "udta 3gpp string ukrainian",
    src: &Udta3GppString{
        AnyTypeBox: AnyTypeBox{Type: StrToBoxType("titl")},
        Language:   [3]byte{0x15, 0x0b, 0x12}, // "ukr": u=21, k=11, r=18
        Data:       []byte("Новини"),           // UTF-8 байти
    },
    dst: &Udta3GppString{
        AnyTypeBox: AnyTypeBox{Type: StrToBoxType("titl")},
    },
    bin: []byte{
        0, 0x00, 0x00, 0x00,  // version + flags
        0x6A, 0x92,           // Language: "ukr" в упакованому вигляді
        0xD0, 0x9D, 0xD0, 0xBE, 0xD0, 0xB2, 0xD0, 0xB8, 0xD0, 0xBD, 0xD1, 0x96, // "Новини"
    },
    str: `Version=0 Flags=0x000000 Language="ukr" Data="Новини"`,
    ctx: Context{UnderUdta: true},
},
```

**Запустіть тест**:
```bash
go test -v ./mp4 -run TestBoxTypes3GPP/udta_3gpp_string_ukrainian
```

## ⚠️ Поширені помилки та як їх уникнути

| Помилка | Симптоми | Рішення |
|---------|----------|---------|
| Неправильне кодування мови | `Language="und"` замість `"ukr"` | Використовуйте формулу: `код = (літера - 'a' + 1)` |
| Забути вирівнювання з `Pad` | Зсув даних на 1 біт → помилка парсингу | Пам'ятайте: 1 біт Pad + 15 біт Language = 2 байти |
| Неправильний контекст | Бокс парситься не як `Udta3GppString` | Завжди передавайте `ctx: Context{UnderUdta: true}` |
| Не перевіряти позицію курсора | "З'їдання" наступного боксу | Додайте перевірку `r.Seek(0, io.SeekCurrent)` |
| Ігнорувати `Stringify` у логах | Важко дебажити складні бокси | Використовуйте `Stringify()` для логування структур |
